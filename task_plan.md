# 任务计划：补齐 1/2/3/4 前后端闭环

## 目标
- 1) 新增租户管理前端页面，完成租户成员邀请/角色调整/移除闭环
- 2) 在运营中心补齐未覆盖后端能力（指标、配额、评测、告警分发、安全基线、SLA、Runbook）
- 3) 在治理页补齐保留策略清理与 PII 脱敏能力
- 4) 将 2FA 从“偏好开关”升级为真实 TOTP 流程（后端接口 + 前端交互 + 登录校验）

## 阶段
1. completed - 对齐后端接口与前端缺口，明确实现与测试范围
2. completed - 先写失败测试（API client + 关键页面/流程）
3. completed - 实现后端 MFA(TOTP) 接口与登录校验
4. completed - 实现前端租户管理与 Ops/Governance 扩展页面能力
5. completed - 实现前端 2FA 真流程（设置、启用、停用、登录挑战）
6. completed - 回归验证（前端测试/构建，后端测试）
7. pending - 分批提交与汇报

## 风险
- 登录流程变更涉及兼容老 token 行为，需要保证未启用 2FA 用户无感
- 运营中心功能面较大，优先做可用闭环，避免一次性重设计导致不稳定
- 前后端同步改动较多，必须以测试收敛
